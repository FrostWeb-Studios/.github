# Contributing to FrostWeb Studios

Thanks for your interest in contributing.

Most of FrostWeb's code lives in private repositories operated by the studio team. For our **public plugins, docs, and tools**, we welcome:

- Bug reports
- Reproducible test cases
- Targeted bug-fix pull requests
- Documentation improvements

## Before You Start

- Search existing issues — your problem may already be tracked
- For non-trivial changes, open an issue first to discuss the design
- Read [SECURITY.md](SECURITY.md) before reporting anything security-related (do not file public issues for vulnerabilities)

## Pull Request Process

1. Fork the repository and create a topic branch from `main` (or the repo's default branch)
2. Make your changes — keep them tightly scoped
3. Match the surrounding code style; don't reformat unrelated lines
4. If the repo has tests, add or update them to cover your change
5. Open a PR against the upstream default branch with:
   - A clear title
   - A description explaining **why**, not just **what**
   - Reference any related issue (e.g. `Fixes #42`)

PRs that change unrelated files, do speculative refactors, or add unrequested features will likely be closed. Keep diffs focused.

## License & Sign-Off

Contributions are accepted under the license of the receiving repository. By submitting a PR you affirm that you have the right to contribute the code and agree to license it under those terms.
